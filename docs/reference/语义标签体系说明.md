## 二级子类：AI 自拟（开放），代码中有预设参考列表

### 来源：不是 PCIC 官方，是我（Claude）在建系统时手写的

`_TAG_SUB_EXAMPLES` 里的二级列表**不是从 PCIC 官方文档里提取的**，而是我在设计标签系统时，根据对每个一级大类常见子领域的理解，**手写的一组示例/参考**。

PCIC 官方原文的 Nociones Específicas 清单本身是一个**多层的词条清单**（大类 → 若干子概念 → A1–C2 分级的具体词条），结构比 2 层深。我在实现时做了**简化**：

- 取了一级作为封闭的顶层
- 手写了一组"看起来合理"的二级示例放在代码里作为参考
- 提示词里，这些二级**不是强制约束**——AI 可以自拟

## 提示词里的实际约束

### 1. AI 新词条（`buildAIEntryPrompt`）

传给 AI 的是**一级完整列表 + 每个一级下前 4 个二级示例**：

```
个体·身体维度：身体部位/生理特征/身体动作/外貌
个体·感知与心理：情绪/性格/感官知觉/思维意识
...
```

硬约束是：
- `tags` 格式：`「一级大类 > 二级子类」`
- 一级必须是上面 20 个之一
- 禁止词性/语法标签
- 同条目多标签不得同属一大类；1-3 个

**二级不强制从预设列表选**——AI 可以自拟。预设列表只是给 AI 提供参考方向。

### 2. AI 补全（`buildCompletionPrompt`）

```
tags: 语义标签数组，必须是「一级大类 > 二级子类」两级格式
（一级必须是下列 PCIC 官方 20 类之一，二级自拟具体子类）
```

明确写"二级自拟具体子类"——**开放的**。

### 3. AI 标签重组（`buildTagRestructurePrompt`）

```
优先建二级子类（如「饮食 > 甜品」「旅行·住宿·交通 > 交通工具」）
最大深度 2 层（根→子类→叶子），叶子到根 ≤ 2 跳
```

二级名称**完全由 AI 根据现有标签数据推断**，不强制对齐预设列表。

## 总结

| | 一级 | 二级 |
|---|---|---|
| 来源 | PCIC 官方 Nociones Específicas | Claude 手写参考 + AI 自拟 |
| 约束 | 封闭：20 个之一 | 开放：AI 自由命名 |
| 代码预设 | `TAG_TAXONOMY_ROOTS` | `_TAG_SUB_EXAMPLES`（仅作提示词参考） |
| 能否新增 | ❌ 不能 | ✅ 能（用户手动 / AI 自动） |

### 潜在问题

当前设计下，二级标签是**完全自由的**——AI 每次生成的二级名称可能不一致（比如同一个概念这次叫"甜品"下次叫"甜点"），导致标签碎片化。如果后续想要更严格的语义一致性，可以：

1. 把 `_TAG_SUB_EXAMPLES` 扩展为一套**完整封闭的官方二级清单**（从 PCIC 原文的官方子概念中提炼）
2. 提示词从"二级自拟"改为"二级必须是下列之一"

这是一个需要在「灵活性」和「一致性」之间权衡的选择。
