# 词根作为独立语言单位（词根详情面板）

- 日期：2026-07-06   状态：已采纳

## 背景

韩/日姊妹项目里，"词根"（韩语初声下的分组、日语的音根）只是**导航分组**——没有独立实体，没有自己的释义/标签，点词根只是为了筛选/定位词条。

但西班牙语的词根（hel-/gel-/am-/struct-）本身是**有意义的语言单位**：
- 有核心释义（hel- = 寒冷、凝固）
- 有拉丁语源（gelatus）
- 经历特定音变（g→h）
- 衍生出整个词族（helado/helar/helada/hielo/heladero）
- 有跨语言 cognate（意大利语 gelato、英语 gelatin）

学习者看一个新词时，最想知道的往往是"它的词根是什么、什么意思、还有哪些同根词"。所以词根需要自己的"主页"。

## 抉择

- **选项 A**：沿用韩/日模型，词根只是导航分组，点词根只筛选/定位词条。
- **选项 B**：词根作为独立实体，点词根显示「词根详情面板」（聚合释义/拉丁源/音变/cognate/词族）。
- **选项 C**：词根作为独立 entry（type='词根'），有自己的完整字段。数据冗余，词根和词条两套结构。

## 决定

采用 **B**（聚合视图，不引入独立 entry 类型）。

- `getRootMeta(root)`：从该词根下所有词条聚合元信息（取第一个有值的词条的 rootMeaning/latinOrigin/analysis/etymology/cognates）。
- `renderRootDetail(panel, root)`：渲染词根聚合视图——词根释义 + 拉丁源 + 语义分析 + 词源 + cognate + **同根词族列表**（每个词可双击进词条详情）+ 语义标签聚合。
- `selectRoot`：单击设 highlightRoot + **清空 selectedEntryId** → `renderDetail` 切到词根详情分支。
- `renderDetail`：`if (!selectedEntryId && highlightRoot) renderRootDetail(...)`。

## 后果

- 交互层次：单击词根 → 词族总览；双击词条 → 该词条详情（形态拆解可视化）。
- 词根信息无需重复维护——从词条聚合，词条改了词根视图自动更新。
- 不引入新 schema（选项 C 的代价），复用现有 entry 结构。
- 后缀/前缀（-ado）作为特殊词条（pos='后缀'），有自己的释义页，也走同一详情面板。

**这是西语项目与韩/日的根本架构差异**。韩/日的"词根"无此面板，因为它们的"词根"（初声/音根）主要是发音单位，语义聚合弱；西语词根是语义/词源单位，聚合强。
