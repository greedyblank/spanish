# 词根/词缀作为「真实条目」，词根详情不再借用派生词

- 日期：2026-07-11   状态：已采纳

## 背景

原设计里，词根（如 `hel-`、`am-`）**没有自己的数据条目**——词根详情面板的释义/分析/词源/同源词，是从"挂在该词根下的第一个词"里 `entries.find(e => e.etymology)` 借来的。而词缀 `-ado` 却已经是个独立词条（pos=后缀）。这套「借用」逻辑有严重问题：

- 用户报告 bug（#3）：`bienvenidos` 同属 `bien-` 和 `ven-` 两个词根，点开 `bien-`/`ven-` 词根详情，显示的却是 `bienvenidos` 这个词的语义分析、词源、跨语言同源——而不是 `bien-`（好）/`ven-`（来）这两个词根**各自固有**的含义。
- 词根/词缀无法编辑、补全、删除自己的信息（#4 要求它们要能）。

## 抉择

- **选项 A**：只词缀成条目（延续 `-ado`），词根用单独的轻量元数据表存释义/分析。改动小，但两套存储、词根不能复用词条的编辑/补全/删除 UI。
- **选项 B**：词根/词缀都成为**真实条目**（pos=词根/前缀/后缀），有自己独立存储的释义·分析·词源·同源词，复用词条的编辑/补全/删除。主词条列表默认隐藏它们（避免污染），可一键切换显示。
- **选项 C**：都成条目且都显示在主列表（与 `-ado` 完全一致）。最简单但主列表被大量词根/词缀条目稀释。

## 决定

采用 **B**（用户在澄清中选「成条目但主列表默认隐藏」）。

- `isRootEntry(e)` = pos ∈ {词根, 前缀, 后缀}；`findRootEntry(root)` 找 word===root 的自身条目。
- `renderRootDetail` 的释义/分析/词源/同源词一律取 `meta.self`（自身条目），**绝不 fallback 到派生词**。无自身条目时显示⚠提示 +「建立词根条目」按钮（`createRootEntry` 新建）。
- `getVisibleEntries` 默认过滤掉 `isRootEntry`（`state.showRootEntries=false`）；「词根条目」按钮 + `toggleShowRootEntries` 切换。搜索时不隐藏（方便找词缀）。
- 词根/词缀详情提供 编辑/AI补全/删除 按钮；补全 prompt 对词根/词缀条目走「固有含义」规则（见 ADR 0006 与 CHANGELOG 西5 #3）。

## 后果

- 词根/词缀有了稳定的自身语义，不再被某个偶然排第一的派生词带偏。
- 词族列表里，词根/词缀自身条目与派生词分离（详情面板顶部是词根本身，下方「派生词族」不含自身）。
- 代价：一个词根要有准确释义，需要它有自身条目；旧数据中大量词根无自身条目，详情走「派生词推断 + ⚠提示」过渡，可用「建立词根条目」或 AI 补全逐步补齐。
- 统计口径：顶部「词条数」含词根/词缀条目，主列表可见数可能少于它（隐藏了词根条目），属预期。
